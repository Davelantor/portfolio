# Project Overview

You are a senior UI designer and frontend developer.
Build premium, dark-themed interfaces.
Use subtle animations, proper spacing, and visual hierarchy.
No emoji icons. No inline styles. No generic gradients.

Claude Code to behave the way I want.
Each feature does one thing, the code is easy to follow, and the app is
easy to run locally and deploy.

## Mission
Build a premium portfolio website for a graphic designer / UI-UX designer.
The site should feel cinematic, minimal, high-fidelity, and presentation-grade.

## Goals
- dark-mode-first
- modern, elegant, and polished
- clear case studies and strong storytelling
- responsive, accessible, and fast
- visually refined, never generic

## Design direction
Use:
- sleek dark surfaces
- strong typography
- generous spacing
- clean grid layouts
- subtle glow, depth, and gradients
- calm, premium motion

Aim for the feeling of a luxury tech presentation: precise, stylish, and confident.
Do not copy any specific brand, layout, or asset.

## Preferred stack
Default to:
- Next.js
- TypeScript
- Tailwind CSS
- Framer Motion
- clean reusable UI components

Prefer stable, modern tools over experimental ones.

## UX rules
- clarity first
- simple navigation
- strong hierarchy
- curated sections, never crowded
- motion should guide attention, not distract
- mobile experience must stay elegant and readable

## Visual rules
- dark theme by default
- restrained accent colors
- premium sans-serif typography
- large hero sections
- polished case study layouts
- crisp imagery and tasteful mockups

Avoid:
- clutter
- loud neon palettes
- overdone glassmorphism
- gimmicky animation
- template-like design

## Case studies
Each case study should show:
- overview
- role
- challenge
- process
- final UI
- outcome

Do not invent fake metrics, clients, or testimonials.

## Copy style
Write in a tone that is:
- concise
- calm
- modern
- confident
- tasteful

Avoid buzzwords and generic marketing fluff.

## Workflow
When working:
1. inspect the current repo and UI
2. identify the highest-impact improvement
3. keep the visual system consistent
4. build reusable components
5. self-review for polish, responsiveness, accessibility, and performance

## Done means
A task is done only when the result is:
- premium-looking
- responsive
- accessible
- maintainable
- visually cohesive
- portfolio-worthy
