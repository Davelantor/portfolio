NDA · Mobile · Disaster Relief OPSEC · 2018

Secondary Seismic
Activity Detection

A mobile app designed for use in the hours and days after a major earthquake, when aftershock risk is highest and connectivity is unreliable. The core mechanism is a consensus model: each device listens to its onboard sensors, but an alarm only triggers when a cluster of nearby devices agrees the pattern is regional shaking — not footsteps, generators, or local vibration. The UX challenge was building trust without false precision, and designing for stressed users operating with shaky hands in noisy, low-connectivity environments.

Key contributions

  • Owned end-to-end product design — from product framing and trust model through IA, flows, and high-fidelity UI
  • Designed glanceable monitoring, mesh transparency, and stress-ready alarm interactions for high-stakes field use
  • Conducted field research and stakeholder interviews; led product scoping to a focused, testable MVP
  • Engineered UX for degraded states — low battery, poor connectivity, and "unreachable" devices treated as first-class product conditions

B2B Portal · Web · UX Concept · 2026

Laboratory Sample
Tracking System

A B2B customer portal for businesses that ship physical samples to a lab for analysis. The core design problem: typical portals force users into detail pages to understand order status, bury test selection in long flat lists, and make repeat orders unnecessarily manual. I rebuilt the experience around a status-first dashboard, a two-level taxonomy-based filter that mirrors how users actually think, and a duplicate-order flow that cuts repeat work down to seconds.

Key contributions

  • Owned end-to-end UX — product framing, IA, order lifecycle status model, and all core flows
  • Designed status-forward order cards where progress strip and next action replace the traditional table row
  • Built a full create-order wizard covering test selection, sample logging, shipping label printing, and review
  • Reduced order creation time by 80% and repeat order time by 300% through the duplicate feature

Bimar

Enterprise Desktop · Logistics Compliance · Project Lead · 2014

Container Storage
Time Tracking System

A local-first desktop application built for customs officers at land border checkpoints, where container movements were still tracked on paper and internet connectivity couldn't be assumed. The core challenge was designing for low-digital-literacy users under time pressure, in shared workstations, in an environment where a missed overdue container meant legal penalties. The solution prioritised status visibility and error prevention over feature richness — making the right action the easiest action.

Key contributions

  • Led the project end-to-end — product vision, scope, stakeholder alignment, UX, frontend development, and pilot rollout
  • Designed a status-first UI that eliminated manual date calculation and surfaced compliance risk passively but persistently
  • Built ISO 6346 container ID validation and date constraints directly into the entry flow to prevent errors at the source
  • Reduced surprise overdue containers from 3–5 per month to zero; estimated task completion improved by over 300%
